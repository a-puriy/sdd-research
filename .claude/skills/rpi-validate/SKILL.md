RPIコマンドプロンプトの妥当性を、策定済みの評価基準に基づいて検証する。

**入力**: 評価対象のコマンド名（`research`, `plan`, `implement`, or `all`）。省略時は `all`。

**運用フロー**:
1. PJ固有のRPIプロンプトを `.claude/commands/rpi/research.md`, `plan.md`, `implement.md` に直接ペースト（上書き保存）する
2. `/rpi-validate` でファイルを参照して評価する

---

## 評価手順

### Step 1: 評価基準と対象プロンプトの読み込み

1. 評価基準を読み込む:
   - `output/prompt-evaluation-guidelines.md` — 全文を読み込む
2. 対象プロンプトをファイルから読み込む:
   - `all` の場合: `.claude/commands/rpi/research.md`, `plan.md`, `implement.md` の全ファイル
   - 個別指定の場合: `.claude/commands/rpi/{指定名}.md`
3. 参照用にフレームワーク実装例を必要に応じて読み込む:
   - Kiro: `.claude/commands/kiro/spec-impl.md`, `spec-design.md`, `spec-requirements.md`
   - SpecKit: `.claude/commands/speckit.implement.md`, `speckit.plan.md`, `speckit.specify.md`
   - OpenSpec: `.claude/commands/opsx/apply.md`, `opsx/explore.md`

### Step 2: コマンド別チェックリストの適用

評価基準の「コマンド別チェックリスト」セクションに定義された各観点について、対象プロンプトを検証する。

**`/research` の場合** — R-1〜R-7 の7観点を検証
**`/plan` の場合** — P-1〜P-10 の10観点を検証
**`/implement` の場合** — I-1〜I-13 の13観点を検証

各観点について以下を判定する:

- **充足**: 評価基準の意図を満たす記述が存在する
- **部分的**: 記述はあるが不十分（具体性不足、条件分岐の欠落等）
- **欠落**: 該当する記述が存在しない
- **過剰**: 評価基準で「不要」と判定された観点が含まれている

### Step 3: アンチパターンチェック

評価基準の「評価軸8: アンチパターンの回避」に記載された4項目をチェックする:

1. テンプレート厳格準拠とフォールバックの矛盾がないか
2. レビュー観点数・質問数の固定がないか
3. 過剰な質問制限（数の上限固定）がないか
4. 変更管理の欠落がないか（特にimplementで重要）

### Step 4: フレームワーク参照との比較（深さチェック）

各観点について、評価基準に記載されたフレームワーク参照（Kiro/SpecKit/OpenSpecの具体的記述）と比較し、以下を確認する:

- 3FW共通のパターン（必須級）が漏れなく反映されているか
- 1FWのみのパターンについて、採用/不採用の判断が妥当か
- 3FW共通の欠落（変更管理等）がRPIでは補完されているか

### Step 5: 評価レポートの生成

以下の構造で評価レポートを出力する:

```markdown
# RPI プロンプト妥当性評価レポート

**評価日**: YYYY-MM-DD
**対象**: /rpi:research | /rpi:plan | /rpi:implement

## サマリ

| 判定 | 件数 |
|------|------|
| 充足 | N |
| 部分的 | N |
| 欠落 | N |
| 過剰 | N |
| アンチパターン検出 | N |

**総合判定**: PASS / CONDITIONAL / FAIL

## 観点別詳細

### [観点ID] [観点名] — [充足/部分的/欠落/過剰]

**プロンプト記述**: （該当箇所の引用）
**評価基準の要求**: （チェック内容の要約）
**FW参照**: （最も近いフレームワーク実装例の引用）
**判定理由**: （なぜその判定か）
**改善案**: （部分的/欠落の場合のみ）

（全観点について繰り返す）

## アンチパターンチェック

| # | チェック項目 | 結果 |
|---|------------|------|
| 8.1 | テンプレート準拠/フォールバックの矛盾 | OK / NG |
| 8.2 | レビュー件数の固定 | OK / NG |
| 8.3 | 質問数の上限固定 | OK / NG |
| 8.4 | 変更管理の欠落 | OK / NG |

## 改善アクション一覧

（欠落・部分的・アンチパターンNGの項目を優先度順にリスト化）

1. **[必須]** [観点ID] — 改善内容
2. **[推奨]** [観点ID] — 改善内容
```

---

## 判定基準

- **PASS**: 必須観点がすべて充足 AND アンチパターン検出なし
- **CONDITIONAL**: 必須観点に「部分的」が1-2件あるが「欠落」なし AND アンチパターン検出なし
- **FAIL**: 必須観点に「欠落」が1件以上 OR アンチパターン検出あり

---

## 責務境界

- このスキルは**読み取り専用**の評価を行う。プロンプトファイルを直接編集しない
- 改善案は提示するが、適用はユーザーの判断に委ねる
- 評価基準自体の妥当性は検証対象外（基準の見直しは別途実施）
