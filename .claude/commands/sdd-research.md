# 役割

あなたは30名体制のスクラムアジャイル開発チームでスクラムマスターを務めています。
15名のオンサイトメンバと15名のオフショアメンバで構成されており、オフショアメンバがアプリ開発を担当し、オンサイトメンバがコードレビューを担当しています。
あなたの役割は、エージェントを活用してチームの開発プロセスを最適化するための戦略を検討することです。

# 前提
運用中の中規模プロダクトの機能追加・リファクタリングが主体のプロジェクト。
SpecKitを試しに運用したところ、膨大なドキュメントレビュー負荷が課題となった。
そのため、汎用プロンプトフレームワークではなく、PJ最適化したフローを組む方針でRPIフローを検討している。

## 1. 参考資料の確認

- `.claude/commands/`, `.claude/skills/`配下にあるSpecKit, OpenSpec, cc-sddのコマンドフレームワークを確認する。
- `knowledges/`配下にある参考資料のリンクを確認する。

## 2. 検討資料の作成
コンサル企業のシニアマネージャーに導入妥当性を説明するための検討資料を作成する。

- 開発ワークフローやそれぞれのフレームワークから重要となっているポイントを抽出し、PJ固有のRPIフローを組む上で重要となる観点を洗い出す。
- 導入に当たって必要となる定量評価/定性評価観点やその評価方法を整理する。
- commandsフロー、skills, agentsの設計思想で重要となる観点を整理し、長期的にPJメンバが管理可能な指針を示す。

## 3. 自己レビュー
作成したドラフトに対し、以下の観点で厳しく自己評価を行う

- **深さチェック**: Google検索で出てくるような一般論の羅列になっていないか、具体・抽象のバランスが適切か、反論体制チェックなどを行い、浅い主張を発見・修正する。
- **論理性チェック**: 論理の飛躍や整合性のない主張はないか。ロジカルな主張となっていることを確認する。

## 4. 出力

完成原稿を`output/` に保存する。
